Los códigos de ejemplo representan un típico servicio (como los 'daemon' de linux).

Cuando se recibe una petición de un cliente, el servidor genera:

* en unos casos un proceso hijo,

* en otros casos un thread dedicado,

que atiende la petición del cliente y genera la respuesta.


Los códigos pueden fallar de diferente forma: división por cero, acceso a espacio de memoria prohibido, etc.


Comprueba cómo se comportan las diferentes implementaciones del servidor.


Los códigos con nombre terminado en _forkrate y _threadrate realizan muchas llamadas a fork() y pthread_create() respectivamente para que tengáis una medida del coste de cada tipo de llamada al sistema.
